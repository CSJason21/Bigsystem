## 第一步：什么是"训练一个模型"

假设你要训练一个 AI 模型来**识别诈骗交易**。模型本质上就是一个数学公式，里面有一大堆**参数**（可以理解成"旋钮"）。

```
模型 = 一个包含几百万个参数的数学公式

参数就像这样：
  旋钮1 = 0.73
  旋钮2 = -0.15
  旋钮3 = 0.42
  ...（几百万个）
```

**训练的过程就是不断调整这些旋钮**，让模型的预测越来越准。

## 第二步：什么是"梯度"

"梯度"就是**告诉你每个旋钮应该往哪个方向调、调多少**。

打个比方：

```
你闭着眼睛在山上，想走到谷底（最低点 = 模型最优）

你用脚探一下：
  - 左脚往前，地面升高了 → 梯度说"别往这走"
  - 右脚往前，地面降低了 → 梯度说"往这走"

梯度 = 每个参数（旋钮）应该调整的方向和幅度
     = 一组数字，和参数一一对应

比如：
  参数1 当前值 0.73，梯度 -0.05 → 说明参数1 应该减小一点
  参数2 当前值 -0.15，梯度 +0.12 → 说明参数2 应该增大一点
  ...（几百万个梯度，对应几百万个参数）
```

**一次训练 = 算出梯度 → 按梯度调整参数 → 模型变好一点点**

## 第三步：什么是联邦学习

传统方式：所有数据集中到一个地方训练。但反欺诈场景中，每个银行/机构的数据都涉及隐私，**不能把数据集中起来**。

联邦学习的解决办法：**数据不动，模型动**。

```
传统方式：
  银行A的数据 ──┐
  银行B的数据 ──┼──→ 全部放到一个服务器上训练 ──→ 模型
  银行C的数据 ──┘
  ❌ 数据隐私泄露

联邦学习：
  银行A：在自己家用自己的数据训练 → 算出梯度A
  银行B：在自己家用自己的数据训练 → 算出梯度B
  银行C：在自己家用自己的数据训练 → 算出梯度C
  
  把梯度A、梯度B、梯度C 发送到服务器（不发送数据！）
  服务器把 3 份梯度合并 → 更新全局模型
  把新模型发回给每个人
  
  ✅ 数据始终留在各自手里，只传递梯度
```

映射到你的系统里：

```
10 个客户端（edge-shenzhen-01, edge-shanghai-01, ...）
    各自在本地用自己的数据训练
    各自算出自己的梯度
    把梯度上传到服务器
         ↓
    服务器合并梯度 → 得到新的全局模型
         ↓
    把新模型发回给 10 个客户端
         ↓
    下一轮训练开始（重复 50 轮）
```

## 第四步：合并梯度时出了问题——投毒攻击

恶意节点（比如 `edge-xian-01` 被黑客控制了）故意上传**错误的梯度**：

```
正常客户端上传的梯度：+0.05, -0.03, +0.12, -0.08, ...
恶意客户端上传的梯度：+99.9, -88.8, +77.7, -66.6, ...（故意搞破坏）
```

**如果服务器用 FedAvg（最简单的方式：取平均）来合并**：

```
FedAvg = 把所有梯度简单求平均

10 份梯度的平均 = 9 份正常梯度 + 1 份恶意梯度 的平均

就像 10 个人投票：
  9 个人说"答案是对的"
  1 个人说"答案是错的，而且声音特别大（梯度数值特别离谱）"
  
取平均后 → 结果被严重拉偏 → 全局模型变烂 → 精度从 89% 崩溃到 61%
```

如果有 35% 的节点都是恶意的（10 个里有 3~4 个），FedAvg 就彻底崩了。

## 第五步：你的页面展示的 4 种聚合算法

不同的算法用不同的策略来**抵抗恶意梯度的干扰**：

### FedAvg（联邦平均）—— 最天真
```
做法：所有梯度取平均，一视同仁
比喻：10 个人投票，简单数人头
结果：35% 恶意节点下 → 精度 61% ✗ 崩溃

### Krum —— 排除异己
```

结果：35% 恶意节点下 → 精度 85%
```

### 几何中值 —— 取中间值
```

比喻：10 个人投票，不取平均，取排在中间的那个
结果：35% 恶意节点下 → 精度 82%
```

### Bulyan —— 双重过滤，最鲁棒
```

比喻：10 个人投票 → 先踢掉明显捣乱的 → 剩下的人里再去掉最高最低 → 再取平均
结果：35% 恶意节点下 → 精度 90% ✓ 最强

## 回答你的具体问题

### "客户端是用户的设备吗？"

**不是。** 在你的系统里，客户端就是 10 个边缘节点（`edge-shenzhen-01` 等），是机房里的服务器。之所以叫"客户端"，是因为在联邦学习的术语里，参与训练的各方都叫 client（客户端），相对于做聚合的 server（服务器）而言。但它们都是**算力基础设施**，不是个人设备。

**对，就是这样。** 边缘节点就是负责训练的。每个边缘节点：

**需要算力，但比训练本身小得多。**

| 步骤                 | 在哪跑                | 算力消耗                           |
| -------------------- | --------------------- | ---------------------------------- |
| 每个边缘节点本地训练 | 10 个 edge 节点各自跑 | **大**（GPU 密集）                 |
| 服务器聚合 10 份梯度 | 超算中心或聚合服务器  | **小**（主要是数学运算，CPU 就够） |

聚合做的事情其实就是**对 10 组数字做统计运算**（求平均、求距离、排序、筛选），和训练模型比起来计算量很小。
