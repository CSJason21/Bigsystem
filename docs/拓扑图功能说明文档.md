# 拓扑图功能说明文档

## 1. 功能定位

拓扑图模块用于在“协同预测与预分配”页面中展示算力网络的层级结构、节点运行状态、预测负载和调度链路。它不是单纯的网络连线图，而是把“全国调度中心、枢纽节点、区域中心、省级算网大脑、DC 节点、边缘节点、单节点资源监控”串联起来，帮助用户从宏观拓扑逐级下钻到单个算力节点。

对应页面与组件：

- 页面入口：`src/pages/ComputingSchedule/PredictionAllocation/index.tsx`
- 拓扑图组件：`src/components/TopologyGraph/index.tsx`
- 单节点监控面板：`src/pages/ComputingSchedule/PredictionAllocation/components/NodeMonitorPanel.tsx`
- 前端接口封装：`src/services/api/predictionAllocation.ts`

## 2. 层级视角切换

页面顶部提供“层级视角”选择器，支持按不同业务粒度查看拓扑：

| 视角 | 展示重点 | 典型内容 |
| --- | --- | --- |
| 全国视角 | 展示顶层算力调度网络 | 国家级调度中心、国家级枢纽、区域中心 |
| 区域视角 | 展示单个枢纽区域的下级结构 | 枢纽节点、区域中心、省级算网大脑、DC 节点、边缘节点 |
| 省级视角 | 展示省内算力承载结构 | 省级算网大脑、DC 节点、边缘节点 |
| 单节点视角 | 查看一个具体算力节点的资源情况 | 节点基础信息、CPU/GPU/内存/磁盘、趋势曲线 |

当选择全国、区域、省级视角时，页面展示协同分配拓扑图；当选择“单节点监控”视角时，页面切换为单节点资源监控页面。区域和省级视角还提供“聚焦节点”选择器，用于快速选中某个 DC 或边缘节点并查看详情。

## 3. 拓扑图展示能力

拓扑图使用 G6 渲染，并在预测分配场景下启用自定义的预测节点和动态边样式。

节点展示规则：

- 实心圆表示当前负载，圆内显示当前负载百分比。
- 外层虚线环表示未来短时预测负载，预测负载越高，外环越大。
- 节点颜色按负载程度区分：低负载为绿色，中等负载为黄色，高负载为红色。
- 离线节点以灰色展示，并标记 `Offline`。
- 新接入节点以绿色状态标记，并标记 `New`。
- 节点下方显示节点名称，必要时显示角色或区域副标题。

边展示规则：

- 实线流光边表示当前正在执行或已建立的调度/连接关系。
- 橙色虚线边表示预测或策略侧规划的重定向链路。
- 离线边缘节点不再绘制其承载链路，体现节点下线后的拓扑变化。

画布交互：

- 鼠标悬浮节点时显示摘要卡片，包括节点名称、角色、状态、当前负载、预测负载和区域信息。
- 点击节点时，右侧信息面板展示该节点的详情或调度日志。
- 点击画布空白处取消当前选中节点，右侧面板恢复为拓扑说明。
- 预测拓扑场景下禁用画布缩放拖拽，避免影响层级布局阅读。

## 4. 节点点击详情

用户点击拓扑节点后，右侧信息面板会根据节点类型展示不同内容。

普通节点详情包括：

- 节点状态：`online`、`offline`、`new`
- 当前负载与预测负载
- 节点 ID
- 节点角色
- 所属区域
- IP 地址
- 算力池或资源池
- 带宽
- 时延

如果该节点当前承载运行中的任务，面板会额外显示“执行中任务”列表，包括任务类型、任务名称以及 CPU、内存、GPU 需求。

如果点击的是调度中心、策略代理等调度类节点，右侧面板展示调度日志流，而不是普通节点详情。日志按阶段区分，包含感知、决策、下发、监控等过程，方便说明一次任务分配从资源感知到调度落点的过程。

## 5. 拓扑说明与事件摘要

未选中节点时，右侧面板显示拓扑说明和事件摘要，帮助用户理解图形含义。

说明内容包括：

- 悬浮查看摘要，点击查看详情。
- 外层虚线环表示未来短时预测负载。
- 虚线箭头表示策略代理提前规划的任务重定向。
- 灰色节点表示离线保位，绿色标签表示新接入资源。

事件摘要包括：

- 当前视角下的离线节点数量。
- 当前视角下的新接入节点数量。
- 重定向次数。
- 运行正常、节点离线、新节点接入等事件卡片。

## 6. 单节点资源预览

切换到“单节点监控”视角后，页面展示具体算力节点的资源预览和趋势分析。

基础信息区展示：

- 节点名称、节点 ID、IP 地址
- 所在区域
- 节点角色
- 算力池
- CPU/GPU 架构
- GPU 数量
- 网络指标：时延、抖动、丢包率、带宽

资源概览区展示：

- CPU 利用率趋势
- 内存利用率趋势
- GPU 使用情况
- GPU 显存占用
- 磁盘占用
- 最近一段时间的资源曲线

在独立的 `NodeMonitorPanel` 组件中，还支持通过节点下拉框切换节点，并展示更细的监控面板：

- 总 CPU、系统 CPU、用户 CPU
- GPU 使用率、显存已用/总量、显存占比
- 内存已用/总量
- 磁盘已用/总量、可用磁盘
- CPU、GPU、内存历史趋势曲线

## 7. 数据来源与接口

当前拓扑数据优先从后端接口获取，接口不可用时页面会回退到前端内存构造的模拟数据。

主要接口：

| 功能 | 前端函数 | 后端接口 |
| --- | --- | --- |
| 视角列表 | `getPerspectives()` | `/api/allocation/perspectives` |
| 拓扑图数据 | `getTopologyView(viewId)` | `/api/allocation/topology/view?view_id=...` |
| 调度日志 | `getScheduleLogs(vertexId)` | `/api/allocation/schedule/logs?vertex_id=...` |
| 节点列表 | `getNodes()` | `/api/allocation/nodes` |
| 节点资源快照 | `getNodeDashboard(nodeId)` | `/api/allocation/nodes/{node_id}/dashboard` |
| 节点历史曲线 | `getNodeHistory(nodeId, period)` | `/api/allocation/nodes/{node_id}/history` |
| 节点运行任务 | `getActiveTasks(nodeId)` | `/api/allocation/tasks/active?node_id=...` |

拓扑图数据结构主要包含：

- `nodes`：节点列表，包含节点 ID、名称、类型、坐标、负载、状态和详情数据。
- `edges`：边列表，包含源节点、目标节点、边类型、标签和样式。
- `events`：当前视角事件摘要。
- `rerouteCount`：重定向次数。
- `offlineCount`：离线节点数量。
- `newCount`：新接入节点数量。

## 8. 数据库映射建议

后续接入真实数据库时，建议保持“拓扑节点”和“算力节点”分离建模：

- `dim_topology_vertex`：描述拓扑图上的所有点，包括调度中心、区域中心、服务组件、DC 节点、边缘节点等。
- `dim_compute_node`：描述真正承载任务的算力节点。
- `dim_topology_edge`：描述拓扑静态连接关系。
- `dim_topology_view`：描述全国、区域、省级、单节点等视图。
- `dim_topology_layout`：保存每个视图下节点坐标，保证刷新后布局稳定。
- `fact_topology_runtime_state`：保存拓扑运行态，如在线状态、当前负载、预测负载。
- `fact_topology_event`：保存拓扑事件，用于右侧事件卡片。
- `fact_schedule_log`：保存调度日志，用于点击调度类节点后的日志流。
- `ts_node_metric` 或节点监控时序表：保存 CPU、内存、GPU、磁盘、带宽、时延等单节点资源曲线。

这样可以避免把控制节点、服务节点和真实算力节点混成一张简单节点表，也能支持同一个算力节点在多个拓扑视图中使用不同布局。

## 9. 验收要点

拓扑图功能验收时可重点检查以下内容：

- 能在全国、区域、省级、单节点视角之间切换。
- 全国视角能展示顶层调度中心、枢纽和区域中心。
- 区域、省级视角能展示 DC 节点与边缘节点的层级关系。
- 节点颜色、外环、状态标签能反映当前负载、预测负载、离线和新接入状态。
- 悬浮节点能显示摘要信息。
- 点击普通节点能显示节点详情和运行任务。
- 点击调度类节点能显示调度日志。
- 点击空白画布能取消选中并恢复拓扑说明。
- 单节点视角能展示节点基础信息、网络状态、资源仪表和历史曲线。
- 后端拓扑接口异常时，页面应能回退到模拟数据并继续展示。
