## 三、新增模块 ①  全局安全态势

**位置**:放在「算力网络协同分配拓扑」和「算力资源服务需求预测面板」之间

**对应立项**:2.1.3 量化安全性评估(AHP / D-S / PCA + 4 层评估)

### UI 草图

```
┌─ 全局安全态势 ────────────────────────────────────────────┐
│                                                              │
│   ┃ 总体安全等级 ┃                                          │
│   ┃     B+       ┃    AHP 权重矩阵                          │
│   ┃              ┃    数据 0.30  算法 0.25                  │
│                       网络 0.25  系统 0.20                  │
│   ╱── 数据安全 87 ──╲                                     │
│  ╱                   ╲   D-S 证据合成:0.81                │
│ ┃ 系统 76    算法 82 ┃                                     │
│  ╲                   ╱   PCA 主成分:                       │
│   ╲── 网络安全 79 ──╱     PC1 = 42%(算法风险主导)         │
│                            PC2 = 28%(数据风险次之)        │
│                                                              │
│   [钻取到 数据层 → 陈舒婷]  [钻取到 算法层 → 张鼎功]       │
│   [钻取到 网络层 → 任鹏杰]  [钻取到 系统层 → 当前页]       │
└──────────────────────────────────────────────────────────────┘
```

### 实现思路(怎么轻量做)

- **雷达图**:echarts radar 组件,4 个轴(数据/算法/网络/系统),数据 hardcode 4 个分数。每 30 秒前端随机微调一下数字(±2)模拟"实时评估"
- **AHP 权重矩阵**:就是一个 4×1 的数字表,鼠标悬停 tooltip 解释"层次分析法"
- **D-S 证据合成**:就是一个数字,tooltip 写一行"Dempster-Shafer 证据理论融合多源安全证据"
- **PCA 主成分**:两个百分比 + 一行解释
- **真正难的数学全在 tooltip 文字里**,评审不会要你现场推公式

### 一句话故事

> "系统综合 4 层安全风险得到当前评级 B+,决策大脑根据这个评分动态调整调度策略,比如算法层风险升高时自动切换到 Bulyan 聚合。"

## 四、新增模块 ②  算法策略决策中心

**位置**:放在「4+N+31+X 算力网络协同分配拓扑」下方

**对应立项**:2.1.2 第一个子点 —— 鲁棒强化联邦聚合(Krum / Bulyan / 几何中值)

### UI 草图

```
┌─ 算法策略决策中心 ────────────────────────────────────────┐
│                                                              │
│  当前聚合算法 : Bulyan (RL 自动选)                          │
│                 [手动:○FedAvg ○Krum ○几何中值 ●Bulyan]   │
│                                                              │
│  恶意节点比例 :  ●━━━━━━━━━━━━ 35%   [滑块 0~50%]         │
│  攻击类型     :  [梯度反转 ▼]  (高斯噪声 / 标签翻转)         │
│                                                              │
│  ┌─ 4 算法收敛曲线对比(投毒后)──────────────────┐         │
│  │  Acc 100%                                        │         │
│  │       │       ━━━━ Bulyan      → 90% ✓ RL 选中  │         │
│  │   80% │      ╱━━━━ Krum         → 85%           │         │
│  │       │     ╱━━━━ 几何中值       → 82%           │         │
│  │   60% │  ╱━━━╲                                   │         │
│  │       │ ╱   ━━━━━ FedAvg        → 61% ✗ 崩溃    │         │
│  │   40% └─────────────  轮次 →                     │         │
│  └──────────────────────────────────────────────────┘         │
│                                                              │
│  强化学习决策日志:                                          │
│   16:42 节点梯度异常 35% → 切换 FedAvg → Bulyan             │
│   16:31 攻击强度 < 10% → 切换 Bulyan → FedAvg(省算力)     │
│   16:24 算法层安全分 82 → 维持当前策略                       │
│   ...                                                       │
└──────────────────────────────────────────────────────────────┘
```

### 实现思路(怎么轻量做)

- **4 条收敛曲线**:前端 hardcode 4 个不同长度的数组,Bulyan 收敛到 90%,Krum 85%,几何中值 82%,FedAvg 在第 5 轮后崩溃到 61%
- **滑块控制恶意节点比例**:不用真跑算法,**前端预存 3 套曲线**(20% / 35% / 50% 比例),滑块切换数据即可
- **决策日志**:hardcode 一个模板数组(20 条左右),每 30 秒往上推一条新日志(配合页面顶部虚拟时间)
- **手动切换聚合算法**:radio 按钮选,选了之后高亮对应曲线即可

### 一句话故事

> "这是强化联邦学习的核心 —— 系统检测到 35% 节点有投毒攻击,RL 模型自动从 FedAvg 切到 Bulyan 聚合,精度从 61% 拉回到 90%。决策日志就是 RL 策略的逐步推理过程。"
