### 实现思路(怎么轻量做)

#### Loss / Accuracy 曲线
- echarts 双 y 轴折线
- 全局 Acc + Global Loss 是主线,各 client local Acc 是浅色细线(10 条)
- 数据 hardcode 50 轮训练的数据

#### 数据分布热力图(关键)
- echarts heatmap 组件
- 10 行 client × 10 列 class = 100 个格子
- 颜色深浅表示该 client 上该 class 的样本数
- **滑块控制 α 时,前端切换 3 套预存的热力图配置**:
  - α = 0.1:极度偏斜(每个 client 只有 1-2 个 class 的数据)
  - α = 0.5:中等偏斜(每个 client 有 3-5 个 class)
  - α = 1.0:接近 IID(每个 client 有所有 class)

#### 知识蒸馏效果
- 两个数字 + 一个简单的柱状图(蒸馏前 vs 蒸馏后,每个 client 的准确率)
- 蒸馏前柱子高度参差不齐(σ = 0.18),蒸馏后柱子高度接近(σ = 0.04)

#### 后门攻击检测
- 第 24 轮固定弹一个警告横条
- 告警内容 hardcode

### 一句话故事

> "训练监控员看到联邦训练真的在跑:loss 收敛到 0.42,全局准确率 89%。
>
> 看这张数据分布热力图,10 个客户端的数据极度非 IID(α=0.5),传统 FedAvg 会让某些 client 准确率特别低。系统通过**知识蒸馏自适应**让全局模型在各 client 上的准确率波动从 σ=0.18 降到 σ=0.04 —— 同样的数据,更公平的训练。
>
> 第 24 轮系统检测到 edge-xian-01 这个节点梯度异常,标记可疑攻击,反馈给决策大脑。这就是 4 层安全里的'算法安全层'。"
