## 二、修存量

### 必修

#### 1. 拓扑图客户端节点名重叠

底层 N 个客户端节点名字"客户端-ed客户端-51 客户端-a6 客户端-2a" 全挤在一起。

修法(G6 配置):
```js
nodeLabel: { position: 'bottom', autoRotate: true }
labelCfg: { style: { fontSize: 10, fill: '#666' } }
// 多个节点重叠时,只显示数量徽标
```

或者:同一行客户端 ≥ 3 个时,折叠成"+ N"标签,鼠标悬停展开列表。

#### 2. 边缘节点 GPU 标签字体错

"DI NVIDIA A10" 实际应该是 "4×NVIDIA A10"。

修法:检查 G6 label formatter,或者前端字体引入 Inter / 思源黑体兜底,避免"4×" 被字体渲染成"DI"。

#### 3. 监控数据抖动太大

CPU 使用率 56% → 67% → 50% 一秒一变,看着不真实。

修法:用 OU 过程(均值回归)生成数据,每秒变化幅度 ±5% 以内。简单实现:

```js
let prev = 50;
function tick() {
  // 95% 概率小幅波动,5% 概率较大变化
  const delta = (Math.random() - 0.5) * 6;
  prev = Math.max(20, Math.min(80, prev + delta));
  return prev;
}
```

## 六、跟其他 3 个同学协作的事

### 1. 共用任务 JSON

你这边任务监控的 66 任务,要从共享任务 JSON 读,字段格式:

```json
{
  "task_id": "task-fedtrain-99943",
  "task_name": "联邦训练-图神经网络",
  "type": "联邦训练",
  "status": "running",
  "progress": 42,
  "assigned_node": "edge-shenzhen-01",
  "resource": { "cpu": 8, "memory": 32, "gpu": 1 }
}
```

### 2. 节点命名层级

你这页是中观层(26 超算 + 10 边缘),要跟下面几个层级对得上:

```
国家级 → 4 枢纽 → 8 区域中心(陈圣杰那边)
       ↓
       你这里:26 超算 + 10 边缘 ← 主战场
       ↓
       陈舒婷的 1397 算力节点(挂到你的超算节点下作为子节点)
       ↓
       客户端(你的拓扑底层 + 张鼎功那边)
```

跟陈舒婷一起把"陈舒婷的 1397 个 node-{city}-{seq} 挂到你的某个超算下"这个映射定下来。

### 3. 跨页面跳转

陈舒婷的任务列表"查看分配详情"按钮要能跳到你这页对应任务的详情。

### 4. 任鹏杰你来主导的协作

侧栏菜单"量化安全性评估"占位 → **挪给陈圣杰**(他做全局态势主页),你这页只保留"网络安全层"的下钻入口。
