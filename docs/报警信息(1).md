# 告警信息功能说明

## 一、功能概述

在联邦学习训练过程中，边缘节点可能遭受后门攻击等安全威胁。本功能实现了 **训练监控告警的跨页面共享**：

- **张鼎功（EdgeControl 页面）**：训练监控过程中检测到异常（如后门攻击），告警数据写入数据库
- **陈圣杰（CloudControl 页面）**：从数据库读取告警信息，在决策中心展示系统安全告警日志

当前已有一条种子告警数据：第 24 轮训练中，`edge-xian-01` 节点梯度偏差值达到 3.87，触发后门攻击检测告警。

---

## 二、数据库表结构

告警数据存储在 PostgreSQL 数据库 `computing_network` 的 `dim_alert` 表中：

| 字段 | 类型 | 说明 |
|------|------|------|
| `alert_id` | BigInteger (PK, 自增) | 告警 ID |
| `task_id` | String(64), FK → `fact_task.task_id` | 关联任务 ID |
| `round_number` | Integer | 触发告警的训练轮次 |
| `node_id` | String(64), FK → `dim_compute_node.node_id` | 触发告警的节点 |
| `alert_type` | String(32) | 告警类型（如 `backdoor_detection`） |
| `severity` | String(16) | 严重程度：`critical` / `warning` / `info` |
| `title` | String(200) | 告警标题 |
| `description` | Text | 告警详细描述 |
| `suggestion` | Text | 处理建议 |
| `status` | String(16) | 告警状态：`active`（未处理）/ `resolved`（已处理） |
| `source` | String(32) | 告警来源（如 `edge_control`） |
| `created_at` | DateTime(tz) | 创建时间，默认为当前时间 |

**ORM 模型定义位置**：`server/app/models/unified.py` → `DimAlert` 类

## 四、前端使用方式

### 1. API 调用

前端 API 封装在 `src/services/api.ts` 中：

```typescript
import { getAlerts } from '@/services/api';

// 调用
const res = await getAlerts();
const alerts = res.data.alerts;  // AlertItem[]
```

### 2. AlertItem 类型定义

```typescript
interface AlertItem {
  alert_id: number;
  task_id: string;
  round_number: number;
  node_id: string;
  alert_type: string;    // "backdoor_detection" 等
  severity: string;       // "critical" | "warning" | "info"
  title: string;
  description: string;
  suggestion: string;
  status: string;         // "active" | "resolved"
  source: string;         // "edge_control" 等
  created_at: string;
}
```

### 3. CloudControl 页面已集成

陈圣杰的 CloudControl 页面（`src/pages/FederatedLearning/CloudControl/index.tsx`）已内置告警日志面板：

- 页面加载时自动调用 `getAlerts()` 获取告警数据
- 使用 Ant Design `Timeline` 组件渲染告警时间线
- 按 `severity` 显示不同颜色标记（红色=严重，橙色=警告，蓝色=信息）
- 显示告警标题、描述、处理建议、来源节点和轮次信息

**如果告警数据为空，面板不会显示。**

## 五、如何新增告警数据

直接向 `dim_alert` 表插入记录即可，CloudControl 页面会自动读取并展示。

示例 SQL：

```sql
INSERT INTO dim_alert (task_id, round_number, node_id, alert_type, severity, title, description, suggestion, status, source)
VALUES (
  'T-95822412',            -- 关联的任务 ID（需存在于 fact_task 表）
  30,                       -- 训练轮次
  'edge-shanghai-01',       -- 节点 ID（需存在于 dim_compute_node 表）
  'backdoor_detection',     -- 告警类型
  'warning',                -- 严重程度
  '梯度异常波动告警',         -- 标题
  '节点在第30轮出现梯度异常波动...', -- 描述
  '建议持续监控该节点...',    -- 处理建议
  'active',                 -- 状态
  'edge_control'            -- 来源
);
```

> **注意**：`task_id` 和 `node_id` 必须是 `fact_task` 和 `dim_compute_node` 表中已存在的值，否则会因外键约束报错。
